Engineered for Deep Context

Whether analyzing a full enterprise repository or reviewing a single source script, the engine maps code outlines with surgical precision.

Directory Mapping

Generates visual, box-drawn Unicode ASCII trees representing exact, readable file system structures.

Ecosystem Manifests

Auto-detects and extracts dependencies from manifests: Node, Python, Rust, Go, Ruby, and Java.

Dynamic Routing

Aggregates backend endpoints (Express, FastAPI, Django, Gin) and React Router client views into clean routing tables.

CSS Custom Tokens

Locates and tabulates global CSS variables (`--style-variable: value;`) across all styling configurations.

High-Density Stats

Computes code, comment, and empty lines, details absolute file sizes, and compiles the top 5 heaviest codebase assets.

Single-File Outlines

Parses imports, exported modules, classes, structs, traits, and functions for direct lexical code parsing.

Git Cloner Integration

Instantly clones and analyzes public repositories into isolated sandbox spaces without manual shell configurations.

README Setup Isolation

Uses heuristic keyword scanning filters to automatically locate and parse setup guides and startup instructions.

Live Active Dashboard

Hosts a local HTTP server on port 3000 to display cloned project file structures, metric summaries, and code setup instructions.

How It Works

Three straightforward compilation steps to establish deep codebase blueprints.

1

Install Scanner Skill

Clone the skill repository into scanner_skill/ inside your project directory. The skill dynamically configures its own non-polluting .gitignore footprint.

2

Deep static Sweep

The parser traverses imports, modules, functions, routes, design custom variables, and dependencies to structure blueprints.

3

Context Blueprints

Get three specialized markdown blueprints (readme.md, instruction.md, ui.md) under output_scanner/ or run the port 3000 console.

Vibe Coding Token Optimizer

Save 90%+ of your AI context window limits during Cursor, Tabnine, or Gemini sessions.

❌ The AI Vibe Coding Problem

AI agents (like Cursor, Gemini, Copilot, or Antigravity) are incredibly powerful, but to solve a single issue, they must read your entire codebase multiple times. This redundant scanning rapidly consumes your context window, causing your API token limits to expire extremely quickly.

🚀 The Codebase Blueprint Solution

By running the Codebase Scanner Skill first, your entire project architecture, routing list, design variables, and manifests are compiled into three lightweight, high-density files inside output_scanner/. Simply point your AI agent to read only this directory to provide complete context at a fraction of the token cost!

Practical Walkthrough: Step-by-Step Example

A Clone and navigate inside your target project repository:

git clone https://github.com/octocat/Spoon-Knife.git && cd Spoon-Knife

B Clone the codebase scanner skill directly into the project:

git clone https://github.com/sujoymoulick/scanner-skill.git scanner_skill

C Navigate inside, install dependencies, and execute the scan:

cd scanner_skill && npm install && node scan.js

D Feed high-density output blueprints directly to your AI agent:

The scanner automatically parses the project and outputs three specialized, high-fidelity blueprints under output_scanner/ inside the cloned repo root:

  • output_scanner/readme.md (Complete project metrics and folder topologies)
  • output_scanner/ui.md (API endpoint routing and stylesheet Custom Variables)
  • output_scanner/instruction.md (Step-by-step startup checklists and running roadmaps)

🎉 Success! Simply prompt Cursor or Tabnine to read this folder first, and work with 100% architectural context while saving up to 90% of your token usage!
